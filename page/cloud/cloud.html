Why Erupt Cloud

不只是配置中心

Nacos / Apollo 擅长 yml 键值对；Erupt Cloud 擅长多行结构化数据，二者可以共存、各司其职。

维度 Nacos / Apollo 传统管理后台 Erupt Cloud ✦
配置形态 yml / properties 键值对 接口 + 前端页面 数据库表，支持多行结构化数据
前端工作量 无（纯配置文件） 大量前端开发 零，纯注解驱动
权限控制 用户 / 角色级别 需自行实现 行、列、按钮级精细控制
基础设施依赖 需独立部署 Nacos / Apollo 全套前后端脚手架 复用现有数据库，无额外中间件
面向群体 研发 研发 + 大量前端资源 研发、运营、产品均可操作
Architecture

Server + Node 双端架构

Server 为调度中心，Node 为轻量微节点，接入即注册，不依赖 Nacos，不依赖 Eureka，部署极简。

erupt-cloud-server
Cloud Server

调度中心，负责节点注册、请求路由、权限校验与统一管理台。提供官方 Docker 镜像，无需自行编码，拉起即用。

<dependency> <groupId>xyz.erupt</groupId> <artifactId>erupt-cloud-server</artifactId> <version>${erupt.version}</version> </dependency>
或 Docker：erupts/erupt-cloud-server:latest
Server 文档 →
erupt-cloud-node
Cloud Node

轻量微节点，接入任意 Spring Boot 服务。仅需一个依赖 + 三行配置即可向 Server 注册，并在该服务内获得完整 Erupt 注解能力。

erupt: cloud-node: server-addresses: ['http://cloud-server:9999'] access-token: 9y1pa7eHksHWBTQC node-name: myService

Features

核心特性

DB-backed Config
数据库驱动配置

以数据表替代 yml，后台可视化编辑，节点实时拉取生效，无需重启或维护配置文件。

Zero Frontend
零前端对接

纯 @Erupt / @EruptField 注解描述界面，各服务无需单独搭建前端脚手架。

Full Isolation
资源完全隔离

数据库连接池隔离、代码逻辑隔离，各节点独立运行，天然适合大规模团队协作。

Row / Col RBAC
行列级权限

继承完整 Erupt 权限体系，权限可精细到表、行、列、按钮级别，满足企业级数据安全需求。

Cloud Native
云原生友好

支持 IP 漂移与 k8s svc 映射，不依赖 Nacos / Eureka，Server 端提供官方 Docker 镜像。

Audit Trail
操作审计追溯

任何配置变更均在平台留下痕迹，基于审计可追溯历史操作，比版本回退更精细。

Scenarios

典型应用场景

集群内任意服务都可接入，把「后台管理能力」变成可编排的集群基础设施。

Notification / Message
通知 / 消息服务

通知模板、发送渠道、推送策略等由运营可视化管理，研发无需重复开发后台页面。

Gateway / Algorithm
网关 / 算法服务

限流规则、GPT Prompt、超时配置等复杂多行数据，用表结构比 yml 更易管理与审计。

Multi-team Collab
多团队协作

不同微服务归属不同团队，通过 Cloud Server 统一收口，数据库级别隔离，互不干扰。
